# Recode UI Direction

Recode should feel like a native editor workspace: split views, compact bars, quiet rows, inspector sections, and precise editor surfaces.

## Principles

- Prefer panes, dividers, toolbars, rows, lists, and inspectors over cards.
- Use cards only when the object is genuinely portable or independently actionable.
- Keep the editor canvas visually dominant; surrounding chrome should support it, not compete with it.
- Follow CodeEdit's mental model: navigator, editor, inspector, utility area, status bar.
- Borrow Athas-level density and feature ambition, but keep the visual language calmer and less dashboard-like.
- Use one main accent for selection/action and sparse semantic colors for status.
- Let spacing, alignment, typography, and separators do most of the UI work.

## First Visual Thesis

Refined native cockpit: dark graphite surfaces, thin split-view separators, compact controls, and a luminous editor plane.
